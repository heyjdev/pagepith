# PagePith brand guide

## Name

**PagePith**

`Pith` means the essential substance or core. The name describes the product's job without collapsing into a generic converter label: it keeps the useful core of a webpage. Use `PagePith` as one word with capital P characters.

## Positioning

PagePith strips a web page down to useful text and turns it into clean Markdown, locally.

## Tagline

**Clean pages. Plain Markdown.**

## Voice

- Direct, calm, and technical.
- Prefer concrete verbs: extract, strip, copy, export.
- Say what happens locally; do not imply cloud processing or AI analysis.
- Avoid productivity hype, agent jargon, exclamation marks, and claims that every page can be extracted.

### Short store description

Turn the useful text on the current page into clean Markdown. Copy or export it without ads, images, navigation, or tracking clutter.

### Single-purpose statement

Convert the useful textual content of the active webpage into locally generated Markdown for copying or export.

## Visual idea

A clean white page contains one bold teal core line, with quieter slate text above and below it. It should read first as "page," then as "keep the useful core." The icon uses no letters, browser logos, robot imagery, cloud symbols, or plant imagery.

## Color

| Token | Value | Use |
|---|---|---|
| Ink | `#0B1220` | Main background and body copy |
| Core teal | `#2DD4BF` | Essential-content line and active accents |
| Paper | `#F8FAFC` | Page shape and light surfaces |
| Slate | `#94A3B8` | Secondary text and borders |
| Error | `#EF4444` | Errors only |

Keep the icon flat. Do not add gradients, gloss, or drop shadows.

## Typography

Use the operating system's UI sans-serif stack for the extension. For marketing assets, use Inter when available, otherwise a neutral sans-serif. Use a monospace face only for Markdown samples.

## Icon rules

- Keep a 12.5% safe margin.
- Use a bold silhouette that survives at 16 px.
- Preserve the white page silhouette and teal core line at every size.
- Never place the product name inside the icon.
- Do not add decorative particles, leaves, seeds, or anatomical imagery.

## Store imagery

Screenshots should show the product doing one thing at a time:

1. A noisy article beside the cleaned Markdown result.
2. Preserved headings, links, code, lists, and tables.
3. Copy and export actions.
4. A clear rejection of an artifact-only page.

Use real extension UI and readable samples. Do not fake reviews, ratings, or performance numbers.
